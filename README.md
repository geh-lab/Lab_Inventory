# Lab Inventory Dashboard

공간 기반 실험실/온실 물품 관리용 단일 페이지 웹앱입니다.

## 이번 수정 핵심

- 클릭이 먹지 않는 문제를 방지하기 위해 Firebase SDK를 `app.js` 최상단 정적 import 방식에서 동적 로딩 방식으로 바꿨습니다.
- Firebase 연결이 실패해도 사이드바 탭 전환, 공간 맵 기본 렌더링, 다크/라이트 모드 전환은 먼저 동작합니다.
- `ui-fallback.js`를 추가했습니다. 앱 메인 로직이 로딩되기 전에도 아이콘, 좌측 탭 전환, 테마 버튼이 동작하도록 하는 안전장치입니다.
- 다크/라이트 전환 버튼을 3곳에 배치했습니다.
  - 상단 바
  - 왼쪽 사이드바 하단
  - 오른쪽 아래 고정 버튼
- 브라우저 캐시 문제를 줄이기 위해 `index.html`에서 JS 파일에 버전 쿼리를 붙였습니다.


## 2026-05-27 추가 수정

- 검색 입력 시 전체 화면을 다시 그리지 않고 물품 목록만 디바운스 렌더링하도록 변경해 버벅임을 줄였습니다.
- 물품 데이터는 내부 캐시와 사전 계산된 검색 문자열을 사용해 필터링 속도를 개선했습니다.
- 업로드 사진은 더 작은 크기로 브라우저에서 압축하고 WebP/JPEG로 저장해 Firebase 저장 지연을 줄였습니다.
- 공간 구조를 `대분류 > 중분류 > 소분류`로 입력할 수 있게 변경했습니다. 기존 `권역` 문구는 화면에서 제거했습니다.
- 공간 맵 상단의 온실/실험실 권역 범례 대신 실제 공간 탭을 표시합니다.
- 관리자 화면은 왼쪽 공간 탭을 클릭하면 오른쪽에 해당 공간의 상세 정보와 물품 목록이 표시되도록 개편했습니다.
- 라이트 모드에서 삭제 버튼, 카드 제목, 공간 맵, 배지, 아이콘 대비를 보정했습니다.
- 공간 수정 모달의 사진/입력 영역이 겹치지 않도록 폭과 그리드 레이아웃을 조정했습니다.

## 주요 기능

- Google 계정 로그인 기반으로 동작합니다.
- 관리자 계정만 물품과 공간을 추가·수정·삭제할 수 있습니다.
- 로그인하지 않았거나 관리자 이메일에 포함되지 않은 계정은 읽기 전용으로만 볼 수 있습니다.
- 왼쪽 사이드바 탭을 추가했습니다. 펼침/접힘 상태를 지원하고 접힌 상태에서는 아이콘 중심으로 표시됩니다.
- 대시보드, 물품 목록, 공간 맵, 분류, 관리자 화면으로 나눴습니다.
- 보관 장소 기본값:
  - 온실-창고
  - 온실
  - 사무실
  - 실험실-창고
  - 실험실
  - 공동기기실
  - KT&G
  - 2층
  - 기타
- 관리자 화면에서 공간을 직접 추가·수정·삭제할 수 있습니다.
- 공간별 등록 물품 목록을 관리자 화면에서 바로 확인하고, 물품 상세 화면으로 열 수 있습니다.
- 기본 공간은 삭제 시 완전 삭제 대신 숨김 처리되며, 관리자 화면에서 복구할 수 있습니다.
- 등록된 물품이 있는 공간은 삭제할 수 없습니다. 먼저 해당 물품의 보관 장소를 이동해야 합니다.
- 과제 연관성을 `노지`, `온실`, `모두`로 관리합니다.
- 분류는 체크박스 방식이며, 기본 분류에 `HPLC 분석`, `흡광도 분석`, `해당 없음`을 포함했습니다.
- 분류를 선택하지 않으면 자동으로 `해당 없음`으로 저장됩니다.
- 물품 등록/수정 화면에서 분류를 직접 추가할 수 있습니다.
- 분류 화면에서 관리자 계정으로 분류를 추가·삭제·복구할 수 있습니다. 삭제된 분류는 필터와 등록 화면에서 숨김 처리됩니다.
- 공간 맵에서 각 구역을 클릭하면 해당 장소의 물품을 바로 확인할 수 있습니다.
- 공간 위치 사진을 업로드하거나 이미지 URL로 등록할 수 있으며, 지도 카드와 상세 패널에 표시됩니다.
- 물품 사진은 업로드/선택, URL 입력, 제거를 지원합니다. 업로드 이미지는 브라우저에서 축소 후 Firebase Realtime Database에 저장됩니다.
- 다크 모드와 라이트 모드를 전환할 수 있습니다. 선택값은 브라우저 로컬 저장소에 저장됩니다.
- 기존 Firebase 데이터 경로 `inventory/greenhouse`, `inventory/lab`, `inventory/etc`도 계속 읽을 수 있게 유지했습니다.

## 파일 구조

```text
index.html                 # 화면 구조
styles.css                 # 대시보드/사이드바/맵/카드/관리자 화면/라이트·다크 모드 디자인
app.js                     # Firebase, 관리자 로그인, CRUD, 필터, 맵 UI 로직
ui-fallback.js             # 메인 JS/Firebase 로딩 전에도 탭/아이콘/테마가 동작하게 하는 안전장치
database.rules.example.json # Firebase Realtime Database 규칙 예시
README.md                  # 사용 안내
```

## 관리자 계정 설정

`app.js` 상단의 `ADMIN_EMAILS`에 관리자 Google 이메일을 넣으면 해당 계정만 편집할 수 있습니다.

```js
const ADMIN_EMAILS = ["envlab1315@gmail.com"];
```

관리자 계정을 바꾸려면 이 값을 실제 관리자 Google 이메일로 교체하세요. `ADMIN_EMAILS`를 비워두면 개발·테스트 편의상 Google 로그인 사용자 전체가 임시 관리자로 처리됩니다.

## Firebase Realtime Database 규칙 예시

클라이언트 코드에서 버튼을 숨기는 것만으로는 보안이 완성되지 않습니다. 실제 배포 시에는 Firebase Realtime Database 규칙도 관리자 이메일 기준으로 맞추는 것이 좋습니다.

```json
{
  "rules": {
    ".read": true,
    ".write": "auth != null && auth.token.email == 'envlab1315@gmail.com'"
  }
}
```

## 로컬 실행

화면 클릭과 테마 전환은 `file://`에서도 동작하도록 보강했지만, Google 로그인과 Firebase 저장은 로컬 서버로 실행하는 것을 권장합니다.

```bash
cd Lab_Inventory-main
python3 -m http.server 5500
```

브라우저에서 아래 주소로 접속합니다.

```text
http://localhost:5500
```

Firebase Authentication의 Authorized domains에 `localhost` 또는 `127.0.0.1`을 추가해야 Google 로그인이 정상 작동합니다.
